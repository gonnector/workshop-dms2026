Targeting

Positioning

메시지와 전략 수립

→ 대시보드/히트맵/리코딩에서 코호트별 원클릭 전환

행동 기반 세그먼트 (분석용)

세그먼트명필터 조합분석 목적
구매 완료자방문 URL 포함 /order-complete성공 여정 패턴
장바구니 이탈방문 URL 포함 /cart + 미방문 /checkout이탈 원인 파악
리뷰 탐색자스마트 이벤트 review_read정보 탐색형 행동
검색 유입스마트 이벤트 search_performed검색 vs 브라우징 비교
모바일 사용자디바이스 = 모바일디바이스별 행동 차이
UX 마찰 경험Rage Clicks = 예UX 개선 포인트

비교 분석 조합

코호트 × 행동: cohort_B + review_read → 연구자의 리뷰 패턴
코호트 × 디바이스: cohort_C + 모바일 → 트렌드 팔로워의 모바일 행동
코호트 × 퍼널: cohort_E + 장바구니 방문 → 윈도우 쇼퍼의 이탈 지점

세그먼트 저장 방법

1 상단 "필터" 클릭

2 원하는 조건 조합 설정 (사용자 지정 태그, 디바이스, 이벤트 등)

3 "세그먼트로 저장" 클릭 → 이름 입력

4 이후 상단 "세그먼트" 드롭다운에서 원클릭 적용

저장된 세그먼트는 대시보드 · 히트맵 · 리코딩 · 퍼널 전체에 동시 적용됩니다

Round 1 — Clarity 탐색 가이드

Step 1: 대시보드
세션 수, 페이지뷰, Dead/Rage clicks 확인 → 전체적인 사이트 건강도 파악
Step 2: 코호트별 필터 적용
Filters → Custom Tags → cohort = cohort_A → 대시보드 지표 변화 관찰
다른 코호트로 전환하며 비교
Step 3: 히트맵 Compare
히트맵 → Compare 모드 → cohort_A vs cohort_B → 클릭 패턴 차이 시각 확인
Step 4: 세션 리코딩
코호트별 2~3개 세션 리코딩 관찰 → 실제 행동 패턴 체감

Round 2 — Claude Code 프롬프트

Level 1 — 기본 탐색
"이 CSV 파일의 구조를 분석하고 코호트별 기술 통계를 비교해줘"
Level 2 — 패턴 발견
"코호트별 행동 차이를 시각화하고 각 코호트의 특성을 추정해줘"
Level 3 — 세그먼트 추정
"라벨 없이 클러스터링해서 세그먼트를 추정하고, 실제 코호트와 비교해줘"
Level 4 — STP 전략
"STP 관점에서 세그먼트별 마케팅 액션 플랜을 도출해줘"

여러분의 분석과 얼마나 일치했나요?

현실 세계에서는?

워크숍에서는 cohort_A~E 라벨이 있었지만,
현실에서는 라벨이 없습니다.

그러나 같은 방법론이 그대로 적용됩니다:

Clarity

행동 시그널 수집
(클릭, 스크롤, 체류, 이탈)

Claude Code

라벨 없이 클러스터링
→ 세그먼트 자동 발견

여러분의 사이트에 Clarity를 붙이면, 내일부터 바로 시작할 수 있습니다

핵심 정리

1. 고객 행동 데이터에는 STP 인사이트가 숨어 있다
2. Clarity(무료)로 행동 데이터를 수집하고 탐색할 수 있다
3. Claude Code로 심층 분석과 전략 도출을 자동화할 수 있다
4. 이 조합으로 빠르고 쉽게 액셔너블 인사이트를 얻을 수 있다
