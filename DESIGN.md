## 1. 샘플 이커머스 사이트 설계

### 1.1 사이트 컨셉
**"KOMMA" (코마)** - 한국 라이프스타일 이커머스
- 카테고리: 뷰티, 패션, 리빙, 푸드, 테크
- 타겟: 20~40대 한국 소비자
- 반응형 웹 (모바일/데스크톱)

### 1.2 페이지 구조 및 필수 인터랙션

#### 홈페이지 (`/`)
- 히어로 배너 (슬라이드 3개) → 클릭 트래킹
- 카테고리 내비게이션 바 → 카테고리별 진입 경로 분석
- 추천 상품 섹션 (인기/신상/할인) → 스크롤 깊이 + 클릭률
- 프로모션 팝업 (첫 방문 시 쿠폰) → 팝업 반응률
- 검색바 → 검색 키워드 트래킹
- 최근 본 상품 → 재방문 행동 분석

#### 카테고리 페이지 (`/category/{name}`)
- 필터링 (가격대, 브랜드, 평점) → 필터 사용 패턴
- 정렬 (인기순, 가격순, 최신순) → 정렬 선호도
- 무한 스크롤 or 페이지네이션 → 스크롤 깊이
- 상품 카드 호버/클릭 → 관심 상품 패턴

#### 상품 상세 페이지 (`/product/{id}`)
- 상품 이미지 갤러리 (스와이프/썸네일) → 이미지 탐색 행동
- 옵션 선택 (색상, 사이즈) → 옵션 선택 패턴
- 수량 선택 → 구매 의도 강도
- **장바구니 담기** 버튼 → 전환 핵심 이벤트
- **바로 구매** 버튼 → 즉시 구매 의도
- 찜하기(위시리스트) → 관심도 (구매 보류)
- 상품 설명 탭 (상세정보/리뷰/Q&A) → 탭 전환 패턴, 스크롤 깊이
- 리뷰 섹션 → 리뷰 읽기 깊이, 도움돼요 클릭
- 관련 상품 추천 → 크로스셀링 효과

#### 장바구니 (`/cart`)
- 상품 수량 변경 → 구매 고민 행동
- 상품 삭제 → 이탈 상품 파악
- 쿠폰 코드 입력 → 프로모션 반응
- **주문하기** 버튼 → 체크아웃 진입률
- 예상 배송비 계산 → 배송비 민감도

#### 체크아웃 (`/checkout`)
- 배송 정보 입력 폼 → 폼 이탈 지점 분석
- 결제 수단 선택 (카드/간편결제/무통장) → 결제 선호도
- 포인트/쿠폰 적용 → 할인 민감도
- **결제하기** 버튼 → 최종 전환
- 폼 필드별 에러 발생 → UX 마찰 포인트

#### 주문 완료 (`/order-complete`)
- 주문 확인 정보 → 구매 완료 확인
- 추가 추천 상품 → 포스트 구매 관심도
- SNS 공유 버튼 → 바이럴 잠재력

#### 마이페이지 (`/mypage`)
- 주문 내역 → 재방문 빈도
- 위시리스트 → 관심 상품 관리
- 쿠폰함 → 프로모션 참여도
- 최근 본 상품 → 브라우징 패턴

#### 이벤트/프로모션 페이지 (`/events`)
- 기획전 배너 → 프로모션 반응
- 타임딜 카운트다운 → 긴급성 반응
- 룰렛/쿠폰 이벤트 → 게이미피케이션 참여

### 1.3 핵심 인터랙션 요약 (Clarity 분석용)

| 인터랙션 | 페이지 | 분석 목적 |
|---------|--------|----------|
| 배너 클릭 | 홈 | 프로모션 효과 |
| 카테고리 선택 | 홈/네비 | 관심 카테고리 세그먼트 |
| 검색 | 전체 | 검색 vs 브라우징 세그먼트 |
| 필터/정렬 | 카테고리 | 구매 결정 패턴 |
| 상품 클릭 | 카테고리 | 상품 관심도 |
| 이미지 탐색 | 상품상세 | 상세 탐색 세그먼트 |
| 리뷰 읽기 | 상품상세 | 정보 탐색형 세그먼트 |
| 장바구니 담기 | 상품상세 | 구매 의도 전환 |
| 찜하기 | 상품상세 | 관심 보류 세그먼트 |
| 수량 변경/삭제 | 장바구니 | 구매 고민 행동 |
| 쿠폰 입력 | 장바구니 | 가격 민감 세그먼트 |
| 폼 입력/이탈 | 체크아웃 | UX 마찰 분석 |
| 결제 완료 | 체크아웃 | 최종 전환 |

## 2. Microsoft Clarity 태깅 설계

### 2.1 Smart Events (자동 태그)
Clarity가 자동으로 감지하는 이벤트:
- **Dead Clicks** - 반응 없는 클릭 (UX 문제)
- **Rage Clicks** - 반복 클릭 (사용자 좌절)
- **Excessive Scrolling** - 과도한 스크롤 (콘텐츠 찾기 어려움)
- **Quick Backs** - 빠른 뒤로가기 (기대 불일치)
- **JavaScript Errors** - JS 에러 발생

### 2.2 Custom Tags (수동 태그)

#### 페이지 유형 태그
```
pageType: home | category | product | cart | checkout | order-complete | mypage | event
```

#### 사용자 행동 커스텀 태그
```javascript
// Clarity Custom Tags API 호출 예시
clarity("set", "user_segment", "browser");        // 브라우징만 하는 사용자
clarity("set", "user_segment", "searcher");        // 검색으로 진입하는 사용자
clarity("set", "user_segment", "deal_hunter");     // 할인/쿠폰 중심 사용자
clarity("set", "user_segment", "loyal_buyer");     // 반복 구매자
clarity("set", "user_segment", "researcher");      // 리뷰/상세정보 탐색자

clarity("set", "cart_value", "high");              // 10만원 이상
clarity("set", "cart_value", "medium");            // 3~10만원
clarity("set", "cart_value", "low");               // 3만원 미만

clarity("set", "category_interest", "beauty");     // 뷰티 관심
clarity("set", "category_interest", "fashion");    // 패션 관심
clarity("set", "category_interest", "living");     // 리빙 관심
clarity("set", "category_interest", "food");       // 푸드 관심
clarity("set", "category_interest", "tech");       // 테크 관심

clarity("set", "funnel_stage", "awareness");       // 인지 단계
clarity("set", "funnel_stage", "consideration");   // 고려 단계
clarity("set", "funnel_stage", "decision");        // 결정 단계
clarity("set", "funnel_stage", "purchase");        // 구매 단계

clarity("set", "device_type", "mobile");           // 모바일
clarity("set", "device_type", "desktop");          // 데스크톱
```

#### 커스텀 이벤트 (Custom Events)
```javascript
// 핵심 전환 이벤트
clarity("event", "search_performed");
clarity("event", "product_viewed");
clarity("event", "add_to_cart");
clarity("event", "add_to_wishlist");
clarity("event", "coupon_applied");
clarity("event", "checkout_started");
clarity("event", "purchase_completed");
clarity("event", "review_read");
clarity("event", "filter_used");
clarity("event", "banner_clicked");
clarity("event", "popup_interacted");
clarity("event", "share_clicked");
```

### 2.3 퍼널 설계 (Clarity Funnels)

#### 메인 구매 퍼널
```
홈/카테고리 방문 → 상품 상세 조회 → 장바구니 담기 → 체크아웃 시작 → 결제 완료
```

#### 검색 기반 퍼널
```
검색 수행 → 검색 결과 클릭 → 상품 상세 조회 → 장바구니 담기 → 결제 완료
```

#### 프로모션 퍼널
```
프로모션 배너 클릭 → 이벤트 페이지 조회 → 상품 상세 조회 → 쿠폰 적용 → 결제 완료
```

## 3. Clarity 내장 도구 분석 가이드 (참가자 실습)

### 3.1 Dashboard 분석
- **Sessions / Pages per session / Scroll depth** → 전체 사용자 참여도 파악
- **Dead clicks / Rage clicks / Quick backs** → UX 문제 포인트 즉시 파악

### 3.2 Heatmaps 분석
참가자들이 수행할 태스크:
1. **클릭 히트맵**: 홈페이지에서 어느 영역이 가장 많이 클릭되는지 확인
   - 배너 vs 카테고리 네비 vs 추천 상품 영역 비교
2. **스크롤 히트맵**: 상품 상세 페이지에서 어디까지 스크롤하는지 확인
   - 리뷰 섹션까지 도달하는 비율
3. **영역 히트맵**: 체크아웃 페이지에서 폼 필드별 인터랙션 비교

### 3.3 Session Recordings 분석
참가자들이 수행할 태스크:
1. **구매 완료 세션** 필터링 → 성공적인 구매 여정 패턴 관찰
2. **장바구니 이탈 세션** 필터링 → 이탈 원인 (가격? UX? 정보 부족?) 파악
3. **Rage Click 세션** 필터링 → UX 마찰 포인트 발견
4. **특정 세그먼트별 세션 비교** → 세그먼트별 행동 차이 관찰

### 3.4 Funnels 분석
참가자들이 수행할 태스크:
1. 메인 구매 퍼널 단계별 이탈률 확인
2. **세그먼트별 퍼널 비교**: 모바일 vs 데스크톱, 검색 유입 vs 직접 방문
3. 이탈이 가장 큰 단계 식별 → 개선 우선순위 결정

### 3.5 Filters & Segments
참가자들이 수행할 태스크:
1. Custom tag 기반 세그먼트 필터링 (deal_hunter vs researcher 등)
2. 세그먼트별 KPI 비교 (전환율, 체류시간, 페이지뷰)
3. 고가치 세그먼트 식별

## 4. Claude Code 활용 데이터 분석 설계

### 4.1 Clarity에서 Export 가능한 데이터
- **Session-level 데이터** (CSV export): 세션ID, 시작시간, 지속시간, 페이지뷰 수, 디바이스, 국가, 커스텀 태그 등
- **Heatmap 데이터**: 클릭 좌표, 스크롤 깊이 등 (API 또는 스크린샷)
- **Funnel 데이터**: 단계별 세션 수, 전환율 (수동 기록 또는 API)

### 4.2 Claude Code 분석 시나리오

#### 시나리오 1: 자동 세그멘테이션 (S)
```
프롬프트 예시:
"이 Clarity export CSV 데이터를 분석해서 사용자 행동 기반으로
의미있는 세그먼트를 자동으로 도출해줘.
클러스터링 기법을 적용하고 각 세그먼트의 특성을 설명해줘."

분석 내용:
- K-means 또는 계층적 클러스터링으로 행동 패턴 기반 세그먼트 도출
- 세그먼트별 특성: 평균 체류시간, 페이지뷰, 전환율, 주요 카테고리
- 세그먼트 네이밍 및 페르소나 프로파일 자동 생성
```

#### 시나리오 2: 타겟 세그먼트 우선순위화 (T)
```
프롬프트 예시:
"도출된 세그먼트들을 마케팅 ROI 관점에서 우선순위를 매겨줘.
각 세그먼트의 규모, 전환 가능성, 예상 LTV를 고려해서 분석해줘."

분석 내용:
- 세그먼트별 크기 × 전환율 × 예상 객단가 매트릭스
- BCG 매트릭스 스타일 시각화 (성장성 vs 수익성)
- 타겟 세그먼트 추천 및 근거
```

#### 시나리오 3: 포지셔닝 전략 도출 (P)
```
프롬프트 예시:
"타겟 세그먼트별 최적의 마케팅 포지셔닝 전략을 제안해줘.
행동 데이터에서 드러나는 각 세그먼트의 니즈와 페인포인트를 기반으로."

분석 내용:
- 세그먼트별 주요 관심 카테고리, 가격 민감도, 콘텐츠 소비 패턴
- 세그먼트별 맞춤 메시지, 채널, 프로모션 전략
- A/B 테스트 가설 자동 생성
```

#### 시나리오 4: 퍼널 최적화 인사이트
```
프롬프트 예시:
"Clarity에서 추출한 퍼널 데이터를 분석해서
단계별 이탈 원인을 추정하고 개선안을 제시해줘."

분석 내용:
- 단계별 전환율 벤치마크 비교
- 이탈 세션의 공통 패턴 분석 (세션 길이, rage click 여부 등)
- 구체적인 UX/마케팅 개선 제안
```

#### 시나리오 5: 종합 STP 리포트 자동 생성
```
프롬프트 예시:
"지금까지 분석한 내용을 종합해서
경영진/마케팅팀에게 제출할 수 있는
STP 분석 리포트를 작성해줘. 차트도 포함해줘."

분석 내용:
- 시각화가 포함된 종합 리포트 (matplotlib/plotly)
- 엑셀/PDF 형태로 export
- 즉시 실행 가능한 액션 아이템 목록
```

### 4.3 Claude Code 활용 베스트 프랙티스

#### Step 1: 데이터 준비
```bash
# Clarity에서 CSV export 후
claude "이 CSV 파일의 구조를 분석하고 데이터 품질을 점검해줘"
```

#### Step 2: 탐색적 분석 (EDA)
```bash
claude "이 데이터의 기술 통계, 분포, 이상치를 분석하고
        시각화해서 보여줘. 특히 세그먼테이션에 유용한
        변수를 식별해줘"
```

#### Step 3: 세그멘테이션 수행
```bash
claude "행동 변수 기반으로 K-means 클러스터링을 수행하고
        최적의 k값을 엘보우 메서드로 결정해줘.
        각 클러스터의 특성을 정리해줘"
```

#### Step 4: 인사이트 도출 및 시각화
```bash
claude "세그먼트별 퍼널 전환율을 비교하는 차트를 만들고
        마케팅 관점에서 의미있는 인사이트를 도출해줘"
```

#### Step 5: 액션 플랜 생성
```bash
claude "분석 결과를 바탕으로 세그먼트별
        마케팅 액션 플랜을 표 형태로 정리하고
        우선순위와 예상 임팩트를 포함해줘"
```

## 5. 페르소나 & 시뮬레이션 데이터 설계

### 5.1 사전 정의 페르소나 (5개 세그먼트)

#### 페르소나 1: "가성비 헌터" (Deal Hunter)
- 행동: 할인 상품 위주 탐색, 쿠폰 적극 활용, 가격 필터 사용
- 전환율: 높음 (할인 시), 객단가: 낮음
- 주요 카테고리: 푸드, 리빙
- 세션 패턴: 짧은 체류, 직접적 탐색

#### 페르소나 2: "신중한 연구자" (Researcher)
- 행동: 리뷰 꼼꼼히 읽기, 상세정보 탭 모두 확인, 비교 탐색
- 전환율: 중간, 객단가: 높음
- 주요 카테고리: 테크, 뷰티
- 세션 패턴: 긴 체류, 높은 페이지뷰, 재방문 多

#### 페르소나 3: "트렌드 팔로워" (Trend Follower)
- 행동: 신상품/인기상품 중심, SNS 공유, 배너 클릭
- 전환율: 중간, 객단가: 중간
- 주요 카테고리: 패션, 뷰티
- 세션 패턴: 모바일 위주, 빈번한 방문

#### 페르소나 4: "충성 고객" (Loyal Buyer)
- 행동: 직접 URL 접속, 마이페이지 활용, 재구매
- 전환율: 높음, 객단가: 높음
- 주요 카테고리: 전 카테고리
- 세션 패턴: 규칙적 방문, 빠른 구매

#### 페르소나 5: "윈도우 쇼퍼" (Window Shopper)
- 행동: 많이 보고 안 삼, 찜하기 많음, 장바구니 이탈
- 전환율: 매우 낮음, 객단가: N/A
- 주요 카테고리: 패션, 리빙
- 세션 패턴: 긴 브라우징, 높은 스크롤 깊이

### 5.2 시뮬레이션 트래픽 비율
